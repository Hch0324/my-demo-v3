# 模拟数据使用说明

本项目已配置模拟 API 数据，方便在没有后端服务的情况下进行开发和测试。

## 配置说明

### 1. 环境变量配置
在 `.env.development` 文件中设置：
```
VITE_USE_MOCK=true
```

### 2. 模拟数据功能

#### 用户相关接口
- **登录接口** (`/user/login`): 支持任意用户名和密码登录
- **获取用户信息** (`/user/info`): 返回模拟的用户信息
- **退出登录** (`/user/logout`): 模拟退出登录

#### 仪表板接口
- **统计数据** (`/dashboard/statistics`): 返回随机的统计数据
- **图表数据** (`/dashboard/chart/{type}`): 支持 line、bar、pie、area 等图表类型

#### 表格接口
- **获取表格数据** (`/table/list`): 支持分页的表格数据
- **删除数据** (`/table/delete/{id}`): 模拟删除操作
- **批量删除** (`/table/batch-delete`): 模拟批量删除
- **更新数据** (`/table/update/{id}`): 模拟更新操作
- **添加数据** (`/table/add`): 模拟添加操作

### 3. 数据生成规则

#### 用户数据
- 用户名：支持任意输入
- 邮箱：根据用户名自动生成
- 头像：使用 DiceBear API 生成随机头像
- 角色：默认管理员角色

#### 表格数据
- 姓名：随机中文姓名
- 邮箱：根据姓名自动生成
- 手机号：随机生成
- 地址：随机中国城市和地址
- 状态：随机 active/inactive/pending
- 创建时间：随机生成

#### 统计数据
- 用户总数：5000-15000 随机
- 订单总数：2000-7000 随机
- 总收入：500000-1500000 随机
- 增长率：10-60% 随机

### 4. 切换真实 API

当后端服务准备就绪后：

1. 修改 `.env.development` 文件：
   ```
   VITE_USE_MOCK=false
   VITE_API_BASE_URL=http://your-backend-api.com
   ```

2. 重启开发服务器：
   ```bash
   npm run dev
   ```

### 5. 自定义模拟数据

要添加新的模拟接口：

1. 在 `src/utils/mock.ts` 中添加数据生成函数
2. 在 `src/utils/request.ts` 的 `mockApiResponse` 函数中添加对应的处理逻辑
3. 在相应的 API 文件中添加接口定义

### 6. 注意事项

- 模拟数据每次刷新页面都会重新生成
- 模拟网络延迟为 300ms，更真实模拟网络环境
- 所有模拟操作（增删改）都会返回成功响应
- 生产环境请确保关闭模拟数据功能
